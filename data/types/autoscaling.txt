autoscaling/ContainerResourceMetricSource/_ContainerResourceMetricSource.yaml
ContainerResourceMetricSource indicates how to scale on a resource metric known to Kubernetes, as specified in requests and limits, describing each pod in the current scale target (e.g. CPU or memory).  The values will be averaged together before being compared to the target.  Such metrics are built in to Kubernetes, and have special scaling options on top of those available to normal per-pod metrics using the "pods" source.  Only one "target" type should be set.
===
autoscaling/ContainerResourceMetricSource/container.yaml
container is the name of the container in the pods of the scaling target
===
autoscaling/ContainerResourceMetricSource/name.yaml
name is the name of the resource in question.
===
autoscaling/ContainerResourceMetricSource/target.yaml
target specifies the target value for the given metric

HorizontalPodAutoscalerSpec describes the desired functionality of the HorizontalPodAutoscaler.
===
autoscaling/HorizontalPodAutoscalerSpec/behavior.yaml
behavior configures the scaling behavior of the target in both Up and Down directions (scaleUp and scaleDown fields respectively). If not set, the default HPAScalingRules for scale up and scale down are used.
===
autoscaling/HorizontalPodAutoscalerSpec/maxReplicas.yaml
maxReplicas is the upper limit for the number of replicas to which the autoscaler can scale up. It cannot be less that minReplicas.
===
autoscaling/HorizontalPodAutoscalerSpec/metrics.yaml
metrics contains the specifications for which to use to calculate the desired replica count (the maximum replica count across all metrics will be used).  The desired replica count is calculated multiplying the ratio between the target value and the current value by the current number of pods.  Ergo, metrics used must decrease as the pod count is increased, and vice-versa.  See the individual metric source types for more information about how each type of metric must respond. If not set, the default metric will be set to 80% average CPU utilization.
===
autoscaling/HorizontalPodAutoscalerSpec/minReplicas.yaml
minReplicas is the lower limit for the number of replicas to which the autoscaler can scale down.  It defaults to 1 pod.  minReplicas is allowed to be 0 if the alpha feature gate HPAScaleToZero is enabled and at least one Object or External metric is configured.  Scaling is active as long as at least one metric value is available.
===
autoscaling/HorizontalPodAutoscalerSpec/scaleTargetRef.yaml
scaleTargetRef points to the target resource to scale, and is used to the pods for which metrics should be collected, as well as to actually change the replica count.
===

autoscaling/MetricSpec/external.yaml

autoscaling/MetricSpec/pods.yaml

autoscaling/MetricSpec/resource.yaml

Such metrics are built in to Kubernetes, and have special scaling options on top of those available to normal per-pod metrics using the "pods" source.
===
autoscaling/MetricSpec/type.yaml
