rbac.authorization.k8s.io/AggregationRule/_AggregationRule.yaml
AggregationRule describes how to locate ClusterRoles to aggregate into the ClusterRole
===
rbac.authorization.k8s.io/AggregationRule/clusterRoleSelectors.yaml
ClusterRoleSelectors holds a list of selectors which will be used to find ClusterRoles and create the rules. If any of the selectors match, then the ClusterRole's permissions will be added
===
rbac.authorization.k8s.io/ClusterRole/_ClusterRole.yaml
ClusterRole is a cluster level, logical grouping of PolicyRules that can be referenced as a unit by a RoleBinding or ClusterRoleBinding.
===
rbac.authorization.k8s.io/ClusterRole/aggregationRule.yaml
AggregationRule is an optional field that describes how to build the Rules for this ClusterRole. If AggregationRule is set, then the Rules are controller managed and direct changes to Rules will be stomped by the controller.
===
rbac.authorization.k8s.io/ClusterRole/rules.yaml
Rules holds all the PolicyRules for this ClusterRole
===
rbac.authorization.k8s.io/ClusterRoleBinding/_ClusterRoleBinding.yaml
ClusterRoleBinding references a ClusterRole, but not contain it.  It can reference a ClusterRole in the global namespace, and adds who information via Subject.
===
rbac.authorization.k8s.io/ClusterRoleBinding/roleRef.yaml
RoleRef can only reference a ClusterRole in the global namespace. If the RoleRef cannot be resolved, the Authorizer must return an error. This field is immutable.
===
rbac.authorization.k8s.io/ClusterRoleBinding/subjects.yaml
Subjects holds references to the objects the role applies to.
===
rbac.authorization.k8s.io/PolicyRule/_PolicyRule.yaml
PolicyRule holds information that describes a policy rule, but does not contain information about who the rule applies to or which namespace the rule applies to.
===
rbac.authorization.k8s.io/PolicyRule/apiGroups.yaml
APIGroups is the name of the APIGroup that contains the resources.  If multiple API groups are specified, any action requested against one of the enumerated resources in any API group will be allowed. "" represents the core API group and "*" represents all API groups.
===
rbac.authorization.k8s.io/PolicyRule/nonResourceURLs.yaml
NonResourceURLs is a set of partial urls that a user should have access to.  *s are allowed, but only as the full, final step in the path Since non-resource URLs are not namespaced, this field is only applicable for ClusterRoles referenced from a ClusterRoleBinding. Rules can either apply to API resources (such as "pods" or "secrets") or non-resource URL paths (such as "/api"),  but not both.
===
rbac.authorization.k8s.io/PolicyRule/resourceNames.yaml
ResourceNames is an optional white list of names that the rule applies to.  An empty set means that everything is allowed.
===
rbac.authorization.k8s.io/PolicyRule/resources.yaml
Resources is a list of resources this rule applies to. '*' represents all resources.
===
rbac.authorization.k8s.io/PolicyRule/verbs.yaml
Verbs is a list of Verbs that apply to ALL the ResourceKinds contained in this rule. '*' represents all verbs.
===
rbac.authorization.k8s.io/Role/_Role.yaml
Role is a namespaced, logical grouping of PolicyRules that can be referenced as a unit by a RoleBinding.
===
rbac.authorization.k8s.io/Role/rules.yaml
Rules holds all the PolicyRules for this Role
===
rbac.authorization.k8s.io/RoleBinding/_RoleBinding.yaml
RoleBinding references a role, but does not contain it.  It can reference a Role in the same namespace or a ClusterRole in the global namespace. It adds who information via Subjects and namespace information by which namespace it exists in.  RoleBindings in a given namespace only have effect in that namespace.
===
rbac.authorization.k8s.io/RoleBinding/roleRef.yaml
RoleRef can reference a Role in the current namespace or a ClusterRole in the global namespace. If the RoleRef cannot be resolved, the Authorizer must return an error. This field is immutable.
===
rbac.authorization.k8s.io/RoleBinding/subjects.yaml
Subjects holds references to the objects the role applies to.
===
rbac.authorization.k8s.io/RoleRef/_RoleRef.yaml
RoleRef contains information that points to the role being used
===
rbac.authorization.k8s.io/RoleRef/apiGroup.yaml
APIGroup is the group for the resource being referenced
===
rbac.authorization.k8s.io/RoleRef/name.yaml
Name is the name of resource being referenced
===
rbac.authorization.k8s.io/Subject/_Subject.yaml
Subject contains a reference to the object or user identities a role binding applies to.  This can either hold a direct API object reference, or a value for non-objects such as user and group names.
===
rbac.authorization.k8s.io/Subject/apiGroup.yaml
APIGroup holds the API group of the referenced subject. Defaults to "" for ServiceAccount subjects. Defaults to "rbac.authorization.k8s.io" for User and Group subjects.
===
rbac.authorization.k8s.io/Subject/name.yaml
Name of the object being referenced.
===
rbac.authorization.k8s.io/Subject/namespace.yaml
Namespace of the referenced object.  If the object kind is non-namespace, such as "User" or "Group", and this value is not empty the Authorizer should report an error.
===
